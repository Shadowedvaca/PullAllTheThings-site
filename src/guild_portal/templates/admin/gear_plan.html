{# ── 5-step workflow ───────────────────────────────────────────────────── #} {% if is_gl %}
{# Step 1 — Sync Loot Tables (Blizzard API — primary data source) #}
Step 1 Sync Loot Tables

Pull item data from the Blizzard API. Stubs all raid/dungeon drop items via the Journal API and all craftable gear via the Recipe API. Run once per season after new content launches, or after a content patch adds new encounters or recipes.

{# Step 2 — Enrich Items (Wowhead + Blizzard metadata) #}
Step 2 Enrich Items

Fetch Wowhead tooltips and Blizzard API metadata for items stubbed in Step 1. Backfills slot type, armor type, tooltip HTML, and item-recipe links. Required before Step 3.

{# Step 3 — Process Tier Tokens #}
Step 3 Process Tier Tokens

Parse enriched token tooltips to detect slot and eligible classes. Populates the tier token translation table and flags suspected junk sources. Must run after Step 2. Safe to re-run.

{# Step 4 — Sync BIS Lists (supplemental — adds BIS recommendations on top of Blizzard data) #}
Step 4 Sync BIS Lists

Scrape u.gg and Wowhead BIS recommendations for all specs. Populates BIS priority entries referencing items already in the item table. Any new items found are enriched inline. Discover URLs first if this is a fresh season. Re-sync Errors retries only failed targets.

Website: Plan Type:
{# Step 5 — Bulk Populate Player Plans #}
Step 5 Bulk Populate Player Plans

Create gear plans for all in-guild characters that don't have one, then fill every unlocked slot from Wowhead Overall. Skips locked slots. Safe to re-run.

{# Step 6 — Rebuild Enrichment (Phase B) #}
Step 6 Rebuild Enrichment

Populate the enrichment.* tables from guild_identity data. Runs all enrichment sprocs in order: items → sources → recipes → BIS entries → trinket ratings → category classification → junk flagging. Safe to re-run after any of Steps 1–4.
